# Skill Inventory Evolution Cadence

## When to Re-Run

| Trigger | Cadence | Why |
|---------|---------|-----|
| Weekly | Every 7 days | AI capabilities shift; your map must keep pace |
| Tool adoption | Within 2 weeks | New agent, IDE, or workflow changes execution/judgment split |
| Role change | Within 30 days | New job, team, or responsibilities = new activity mix |
| Major release | When announced | GPT-5, Claude 4, etc. can shift what's depreciating |

## What Changes Between Runs (Weekly)

- **Appreciating** — May stay stable; add new categories as AI evolves
- **Vulnerable** — Often shrinks (skills get codified) or grows (new AI capabilities create new gaps)
- **Depreciating** — Grows over time; be ruthless
- **Specification gap** — Shrinks as you articulate; re-audit when you can't explain a decision

## Output Evolution

Each run should reference the previous:

```markdown
**Delta from [previous date]:**
- [Skill X] moved from Vulnerable → Depreciating (AI now handles)
- [Skill Y] new Appreciating (orchestration demand increased)
- Forcing function from last run: [status]
```
